# 全应用冷白主题与作者工作区布局优化 Design

## 1. 设计目标

- **视觉目标**：浅色默认呈现冷白效率工具感，深色夜间模式降饱和，减少霓虹色和高亮边框。
- **使用目标**：作者进入作品后优先写作；大纲、角色、设定、伏笔、灵感和 AI 作为上下文辅助，不抢中央编辑区。
- **确认方向**：跟随系统、冷白效率、全应用统一、业务逻辑不变。

## 2. 关键页面与交互

| 页面 / 入口 | 改动目标 | 保持不变 |
|---|---|---|
| 主题系统 | 新增 `system` 默认模式，重做浅/深色 token | 主题仍由 UI store 管理并持久化 |
| 工作区 TopBar | 分为作品导航、核心工具、状态区；窄窗口折叠次要工具 | 书名返回、左右栏开关、工具入口、账号入口 |
| 左侧栏 | 大纲优先，宽度可调整并持久化 | 大纲 / 人物 / 设定 tab、卷章拖拽和上下文菜单 |
| 右侧栏 | 伏笔 / 本章角色 / 灵感 tab 化，一次聚焦一个主面板 | 伏笔状态、角色打开、灵感新增删除 |
| 编辑区 | 浅色白纸感、底部状态栏和浮动按钮降饱和 | Tiptap 编辑、自动保存、摘要、风格、AI 续写 |
| AI 助手 | 悬浮按钮和面板接入新语义色并避让辅助区 | 拖拽、会话、草稿篮、确认写入 |

## 3. 视觉规则

- 主操作：低饱和蓝；品牌点缀：少量金色；成功/同步：绿色；警告：橙色；危险：红色。
- 浅色背景使用冷白、蓝灰和细边框，不使用大面积暖黄纸色。
- 深色背景降低彩色亮度，只保留语义差异，不使用高饱和发光。
- 全局组件优先使用 `--accent-*`、`--success-*`、`--warning-*`、`--danger-*`、`--surface-*` 变量。

## 4. 共享层副作用清单

- `themes.ts` 和 `index.css` 的主题变量会影响所有主题化组件。
- `ui-store` 增加本地 UI 偏好状态，不改变业务 store。
- 工作区布局改动会影响 onboarding spotlight、AI launcher 默认位置和左右栏宽度。

## 5. 验收标准

- AC-LAYOUT-001：首次启动主题为 `system`；系统外观变化时未手动指定浅/深主题的窗口跟随更新。
- AC-LAYOUT-002：浅色/深色下书架、工作区、核心弹窗、AI 助手和底部沙盘没有明显霓虹绿紫蓝主视觉。
- AC-LAYOUT-003：1024 / 1440 / 1920 宽度下顶栏不重叠，左右栏可收起/调整，右侧 tab 和 AI 按钮不遮挡关键入口。
- AC-LAYOUT-004：`npm run build` 通过；新增纯函数单测覆盖主题解析、布局 clamp 和默认 tab 逻辑。
