# 应用更新体验重构 Mission

## 交付基线摘要

- **宿主项目**：证道 Electron 桌面端创作软件
- **项目模式**：change
- **治理档位**：P1
- **风险档位**：medium
- **当前基线 ID**：CR-20260423-013500-update-experience-redesign
- **交付目标**：把现有“后台自动下载 + 标题栏更新按钮”重构为“后台自动检查 + 发现即弹出全局更新页 + 用户手动下载/安装 + 安装失败可回退重试”的完整更新闭环。

## 已确认范围

- 在线更新继续基于 GitHub Releases 和 `electron-updater`。
- 应用仍在打包版后台自动检查更新，但不再自动下载。
- 发现新版本后，立即弹出新的全局“应用设置 / 关于”模态页，展示版本号、发布日期和更新日志。
- 用户点击“下载更新”后才开始下载；下载完成后仍在同一入口里执行“立即安装”。
- 关闭弹框后，书架页和工作区都必须能重新打开“应用设置 / 关于”查看更新状态。
- 若 `quitAndInstall()` 未让应用在 15 秒内退出，UI 必须从“正在安装”回退成可重试状态，并给出明确错误提示。

## 非目标

- 不改动 GitHub Release 生产流程、签名、公证或更新元数据格式。
- 不新增“跳过此版本 / 永久忽略”能力。
- 不把作品级“项目设置”混入应用级更新能力。
