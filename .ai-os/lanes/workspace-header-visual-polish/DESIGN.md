# 工作区头部与核心弹窗视觉收口 Design

## 1. 设计目标

- **视觉目标**：从多彩功能色转为统一、克制、偏正式桌面工具的视觉语言。
- **信息层级**：品牌与当前位置优先，核心工具入口次之，状态信息保持可读但不抢主视觉。
- **确认方向**：克制专业；默认暗色主题使用哑金 / 铜金作为品牌点缀。

## 2. 关键页面与交互

| 页面 / 入口 | 改动目标 | 保持不变 |
|---|---|---|
| 工作区 TopBar | 核心工具入口统一为低饱和按钮，进度条去强发光 | 书名返回、面板开关、横向滚动、标题栏安全区 |
| 角色总库 | 去除 `#111/#333` 和 indigo 主视觉，接入主题变量 | 列表、图谱、时间线、关系创建和删除操作 |
| 设定维基 | 去 purple 主视觉，按钮与选中态改为品牌变量 | 三栏布局、分类、条目编辑和保存删除逻辑 |
| 写作数据中心 | 统一全屏头部、分段控件、指标卡和图表色 | 日/周/月/年数据计算与成就展示 |

## 3. 视觉规则

- 主品牌强调使用 `var(--accent-primary)` / `var(--accent-secondary)`，默认暗色为哑金 / 铜金。
- 工具入口使用同一套 `border + subtle background + muted text`，悬停时只轻微提高边框和背景。
- 状态色保留语义：同步 / 成功 / 进度可用绿色，警告用橙色，危险用红色。
- 组件半径保持 8px 左右，避免过度圆润和高饱和发光。

## 4. 共享层副作用清单

- `index.css` 与 `themes.ts` 的主题变量会影响所有使用 `--accent-*` 的组件。
- `AppBrand` 使用品牌强调色后，书架页和工作区品牌头都会同步变化。
- TopBar 和三个弹窗的视觉类名调整不改变 props、store、IPC 或数据 schema。

## 5. 验收标准

- AC-VIS-001：顶部“总览 / 角色总库 / 设定维基 / 数据”不再以不同高饱和颜色区分，布局在窄窗口下仍可横向滚动。
- AC-VIS-002：角色总库、设定维基、数据中心头部和分段控件视觉一致。
- AC-VIS-003：默认暗色主题品牌色为哑金 / 铜金，页面不再呈现明显 AI 紫蓝绿风格。
- AC-VIS-004：`npm run build` 通过；若测试无法全量通过，说明阻塞并保留至少一项项目原生验证证据。
