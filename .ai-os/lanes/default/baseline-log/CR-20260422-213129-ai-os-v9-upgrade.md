# 变更请求：AI-OS v9 全量升级

- **变更标题**：AI-OS framework 从 v7.2.2 升级到 v9.0.0
- **提出时间**：2026-04-22 21:31:29 Asia/Shanghai
- **变更原因**：上游 `royeedai/ai-os` 已发布 v9.0.0，当前项目 AI-OS 元数据仍为 v7.2.2，且新版 doctor 检出根共享 Mission 与部分 lane 扩展工件缺失。
- **优先级**：P1

## 变更内容

- 采用官方 v9 canonical layout：根共享 `.ai-os/MISSION.md` / `.ai-os/memory.md` + `.ai-os/lanes/default/` 默认交付线。
- 替换根 `AGENTS.md` 为 v9 交付宪法。
- 移除旧 `.agents/` workflow / skill / policy / reference 体系，以及 `.cursor/skills`、`.cursor/rules` 派生文件。
- 将旧 `.ai-os/project.md`、`.ai-os/CONVENTIONS.md` 与各 lane `acceptance.yaml` 迁入 v9 对应工件。
- 补齐所有现有 lane 的 v9 扩展工件，不修改业务源码。

## 影响分析

| 维度 | 是否受影响 | 说明 |
|------|------------|------|
| MISSION | 是 | 新增根共享 `.ai-os/MISSION.md`；各 lane Mission 保持交付基线职责。 |
| baseline-log | 是 | 本记录作为升级审计入口。 |
| spec | 否 | 不改变产品需求 spec。 |
| tasks | 否 | 不改变产品任务清单，仅迁移治理框架。 |
| tests | 是 | 需运行 AI-OS doctor、`git diff --check` 与项目原生 `npm run build`。 |
| acceptance | 是 | v9 不再以 `acceptance.yaml` 作为主工件，验收内容并入 lane `DESIGN.md`。 |
| release | 是 | high-risk lane 的发布计划工件需保留或补齐。 |
| memory | 是 | 旧 `CONVENTIONS.md` 稳定约定并入根 `.ai-os/memory.md`。 |
| evals | 是 | 各 lane 补齐 `evals/` 目录，稳定 failure mode 进入 lane 级 guard。 |

## 新增风险 / blocker

- v9 为破坏性升级：旧 slash workflow 和 skill system 被移除，后续依赖根 `AGENTS.md` 宪法规则与 3 个 CLI 命令。
- 当前项目存在多条 active lane，官方 upgrade 只机械规范化 default lane，需要手工补齐其他 lane 的 v9 工件。
- 本轮必须避免触碰现有未提交业务源码改动。

## 后续动作

- 运行官方 v9 `upgrade`。
- 补齐并审查所有 lane 的 v9 工件。
- 运行 v9 `doctor --json`、结构完整性检查、`git diff --check`、`npm run build`。
