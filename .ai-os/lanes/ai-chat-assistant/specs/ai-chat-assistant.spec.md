# AI 创作助手 Spec

## 功能需求

- **FR-AI-001**：工作区右下角提供 AI 创作助手图标，展开后显示非模态对话面板。
- **FR-AI-002**：AI 助手欢迎态展示能力卡：续写、创建章节、审核本章、润色改写、生成角色、生成设定、整理伏笔、剧情节点建议。
- **FR-AI-003**：新增全局账号、全局能力卡、作品 AI 档案、作品能力覆盖、AI 对话、消息和草稿持久化。
- **FR-AI-004**：作品 AI 档案只引用全局账号并保存写作要求；不按作品复制 API Key。
- **FR-AI-005**：能力卡 prompt 必须由全局模板、作品覆盖、作品档案、上下文和用户输入合成。
- **FR-AI-006**：AI 输出写入类结果必须先保存为 `ai_drafts`，由用户确认后应用。
- **FR-AI-007**：旧 BottomPanel 中直接调用旧 AI 逻辑的 `生成章节` / `AI 补全` 入口必须移除，避免绕过全局账号和草稿篮。
- **FR-AI-008**：旧“项目设置 -> AI 配置”只保留迁移说明和进入新配置中心的桥接入口，不再承载 per-book AI provider 表单和 Gemini CLI 登录按钮。
- **FR-AI-009**：Gemini CLI 的状态检测和 Google 登录启动必须出现在“AI 能力与作品配置 -> 全局账号”，并与当前账号 provider 联动。
- **FR-AI-010**：结构化 skill 的草稿解析必须容忍 fenced JSON 或带说明文字的嵌入 JSON，但仍保留 `ai_drafts.kind` 白名单校验。
- **FR-AI-011**：角色、设定、剧情节点、伏笔等结构化草稿必须以可读化字段卡片展示，而不是原始 JSON 字符串。
- **FR-AI-012**：AI 助手面板必须支持拖动位置、拖拽调整大小，并本地持久化位置与尺寸。
- **FR-AI-013**：AI 助手必须支持创建新会话、切换作品内会话、清空当前会话记录。
- **FR-AI-014**：AI 助手输出必须优先流式渲染，token 到达时实时更新 assistant 消息；完成后再持久化消息并生成草稿。
- **FR-AI-015**：AI 对话默认应支持普通问答 / 自动识别意图，不因未显式选择能力卡而默认套用 `continue_writing`；只有用户点击能力卡、选择具体能力或由其他入口带入 skill 时才强制执行该能力卡。
- **FR-AI-016**：AI 会话切换不使用标题栏紧凑原生下拉，改为 AI 面板右侧会话列表；列表支持切换、新建和删除历史会话，标题栏继续保留清空当前会话。

## 接口与数据

- 新增表：`ai_accounts`、`ai_skill_templates`、`ai_work_profiles`、`ai_skill_overrides`、`ai_conversations`、`ai_messages`、`ai_drafts`。
- 新增 preload / IPC：账号、能力卡、作品档案、覆盖、会话、消息、草稿、作品解析 AI 配置相关方法。
- `ai_drafts.kind` 白名单：`insert_text`、`replace_text`、`create_chapter`、`update_chapter_summary`、`create_character`、`create_wiki_entry`、`create_plot_node`、`create_foreshadowing`。
- 旧 `project_config` AI 字段保留；迁移时生成默认全局账号和作品档案引用。

## 验收标准

- **AC-AI-001**：工作区可打开右下角 AI 助手，并从欢迎态选择能力。
- **AC-AI-002**：账号配置是全局资源；作品 AI 档案只保存写作要求和默认账号引用。
- **AC-AI-003**：作品能力覆盖优先生效；无覆盖时继承全局能力模板。
- **AC-AI-004**：AI 写入类结果未确认前不改作品数据，确认后才应用。
- **AC-AI-005**：上下文 chips 显示当前章节、相关角色、伏笔和剧情节点等来源。
- **AC-AI-006**：工作区不再显示旧 `生成章节` / `AI 补全` AI 入口，避免与右下角 AI 助手重复。
- **AC-AI-007**：在全局账号配置里选择 `Gemini CLI` 时，可检测状态并启动 Google 登录；旧项目设置页不再暴露重复登录入口。
- **AC-AI-008**：结构化 skill 返回 fenced JSON 或带说明文字的 JSON 时，AI 助手仍能提取有效草稿并保留白名单校验。
- **AC-AI-009**：角色、设定、剧情节点、伏笔草稿在草稿篮中以可读字段卡片展示，不直接暴露原始 JSON。
- **AC-AI-010**：AI 助手面板支持拖动和拖拽缩放，刷新后仍恢复上次位置与尺寸，且不会被拖出可视区域。
- **AC-AI-011**：用户可在 AI 助手内创建新会话、切换历史会话；新会话初始为空，不影响其他会话记录。
- **AC-AI-012**：清空当前会话前必须二次确认；确认后只删除当前会话消息和当前会话关联草稿，不影响已应用到作品的数据。
- **AC-AI-013**：AI 回复在生成过程中实时渲染，完成后持久化最终 assistant message 并按现有草稿规则进入草稿篮。
- **AC-AI-014**：在未显式选择能力卡时，用户询问“你能为我做什么”等泛问应得到普通助手答复，不创建正文/资产草稿；显式选择“续写正文”等能力后仍按对应 skill prompt 和草稿篮规则执行。
- **AC-AI-015**：点击标题栏会话入口后，右侧弹出会话列表；用户可切换会话、删除历史会话，删除前二次确认，删除当前会话后自动切换或创建空会话。
