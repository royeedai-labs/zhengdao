# 变更请求

- **变更标题**：AI 悬浮入口支持拖动，并移除展开态浮动 X 遮挡
- **提出时间**：2026-04-22 21:48:34
- **变更原因**：用户反馈 AI 对话图标需要可拖动；展开 AI 面板后独立浮动 X 按钮没有必要，且会挡住输入区和发送按钮。
- **优先级**：P1

## 变更内容

- 收起态右下角 AI 圆形入口支持拖动，并把位置持久化到本地 UI 状态。
- 展开 AI 面板后不再渲染独立浮动 X 按钮。
- 展开态关闭 / 收起动作保留在 AI 面板标题栏的收起按钮上。
- 拖动入口时不误触发打开动作；点击未拖动时正常打开 AI 面板。

## 影响分析

| 维度 | 是否受影响 | 说明 |
|------|------------|------|
| MISSION | 否 | 不改变 AI 助手总体目标 |
| DESIGN | 否 | 不改变全局账号、能力卡、草稿篮和会话管理核心设计 |
| tasks | 是 | 新增悬浮入口交互调整任务 |
| tests | 是 | 补充入口位置约束 helper 测试 |
| data | 否 | 不改业务数据、AI 会话数据或草稿数据 |
| release | 否 | 不涉及发布策略 |

## 新增风险 / blocker

- 拖动后的入口必须保持在视口内，窗口尺寸变化时不能丢到屏幕外。
- 拖动结束不能触发打开，普通点击仍应打开。
- 展开态不再有独立 X，因此标题栏收起按钮必须保留。
