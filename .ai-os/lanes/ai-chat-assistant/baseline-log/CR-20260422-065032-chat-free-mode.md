# 变更请求

- **变更标题**：AI 会话默认支持普通问答，不强制绑定写作能力卡
- **提出时间**：2026-04-22 06:50:32
- **变更原因**：用户在 AI 对话中询问“你能为我做什么”等泛问时，意图是了解助手能力，而不是触发“续写正文”等写作 skill。
- **优先级**：P1

## 变更内容

- AI 对话输入默认应进入普通会话 / 自动识别模式，不应因为未选择能力卡而默认套用 `continue_writing`。
- 能力卡仍保留，但其语义改为显式执行：用户点击欢迎态能力卡、在下拉中选择具体能力，或通过入口带入 skill 时，才强制套用该能力卡 prompt 和输出契约。
- 普通会话可以回答能力介绍、配置说明、写作建议、项目资产解释等非写入类问题；默认不创建草稿、不写入作品。
- 如果普通会话判断用户明确要求创建正文、章节或资产，应回复引导用户选择能力卡或进入草稿确认流程，而不是直接写库。

## 新增风险 / blocker

- 需要明确 UI 上“普通对话”和“能力卡执行”的状态，避免用户误以为当前仍在续写模式。
- 普通会话不强制结构化输出，因此草稿生成逻辑必须只在显式 skill 或明确写入流程中触发。
