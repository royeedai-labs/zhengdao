# 变更请求：Gemini 免费使用双通道整合

- **提出时间**：2026-04-21 21:28:47 Asia/Shanghai
- **变更原因**：用户希望当前项目支持免费 Gemini 使用，并同时覆盖 API Key 免费层与 Gemini CLI Google 登录权益路线。
- **优先级**：P1 high-risk
- **基线 ID**：CR-20260421-212847-gemini-free-ai

## 变更内容

- 新增 Gemini API Key 免费层 provider，修复现有 Gemini 设置入口未真正路由到 Gemini REST adapter 的问题。
- 新增 Gemini CLI provider，通过内置 Gemini CLI、主进程服务和 preload IPC 调用 CLI headless 输出。
- 设置页沿用项目级 AI 配置，区分「Gemini API Key 免费层」与「Gemini CLI（Google 登录）」。
- 更新打包配置以携带 Gemini CLI，并记录 Node >= 20 与包体积风险。

## 新增风险 / blocker

- Gemini CLI 依赖体积较大，可能显著增加安装包。
- CLI 登录失败、403、quota、地区限制必须降级为清晰用户提示。
- 若 CLI cwd 或工具权限配置不当，可能扩大本地文件访问面。
