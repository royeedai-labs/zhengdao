# 发布与回滚计划

## 1. 发布目标

- 将仓库顶层授权口径从 `MIT` 切换为 `AGPL-3.0-only`
- 保证 README、包元数据与正式许可证文本一致

## 2. 发布输入

- 用户确认采用 `AGPL-3.0-only`
- 顶层文件改动完成：
  - `LICENSE`
  - `README.md`
  - `package.json`
  - `package-lock.json`

## 3. Smoke Check

- `rg` 检查顶层文件中不再残留旧 MIT 声明
- `npm run build`

## 4. 回滚条件

- 用户确认并非要采用 AGPL，而是其他许可策略
- README / 元数据与许可证文本不一致
- 发现当前工作区已有相关授权改动会被覆盖

## 5. 回滚动作

- 回退本轮顶层许可证与元数据修改
- 保留 `.ai-os` 变更记录，重新发起新的 change-request

## 6. 发布注意事项

- 法律口径变更不等于自动改写历史已发布版本的既有授权事实
- 若后续要对外强调源码开放义务或贡献规则，可追加贡献与版权说明文档
