# AI 操作 UX 原则

本文件记录证道桌面端 AI 操作的通用体验规则。具体业务文案仍以对应组件为准。

## 基本原则

- 让用户先知道一共几步、当前第几步、下一步会发生什么。
- 每一步只问当前真正需要的信息；可选内容不命名为“高级参数”。
- 用快捷选项降低输入成本，同时保留自然语言补充入口。
- AI 等待态说明正在做什么；能确定步数时显示步骤，不显示虚假百分比。
- 涉及写入正文、章节、人物、设定、剧情或伏笔时，必须先展示预览并由用户确认。
- 面向用户使用“方案、预览、确认创建”等可理解词；内部结构名如 package 不直接暴露。

## 新作品起书流程

1. 写灵感：用户可以只输入一句故事灵感。
2. 选方向（可选）：题材、篇幅、人物、背景、风格、边界都可留空。
3. 生成方案：等待态说明整理灵感、补全设定、生成章节人物、校验预览。
4. 确认创建：预览分卷、章节、人物、设定、剧情和伏笔；确认后才创建作品。
